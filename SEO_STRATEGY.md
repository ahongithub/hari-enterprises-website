Ahmedabad/Gujarat and broaden to Pan-India category terms, while staying honest
about the business (supplier/sourcing partner, not manufacturer).

## Search intent tiers
- **Commercial (highest value):** "<product> supplier <city/state>" → product + home pages.
- **Research:** "refractory for <equipment>" → industry pages linking to products.
- **Informational (top-of-funnel):** "how to select / vs / causes of failure" →
  knowledge-centre articles, internally linked to relevant products.

## On-page implementation (already built in)
- Unique `<title>` and meta description per page via `buildMetadata()`.
- One `<h1>` per page; logical `h2/h3` hierarchy.
- Canonical URLs, Open Graph + Twitter cards.
- Semantic HTML5 landmarks; descriptive image `alt` text.
- JSON-LD: Organization, LocalBusiness, Product, Article, BreadcrumbList, FAQPage
  (FAQ schema only on the Why page, which has visible Q&A).
- Dynamic `sitemap.xml` and `robots.txt`.
- Fast Core Web Vitals: static generation, `next/image`, `next/font`, minimal JS.

## Internal linking plan
- Home → product categories, industries, leadership, resources.
- Product pages → related products + category + Request a Quote.
- Industry pages → relevant product categories + sample products.
- Articles → related products + Request a Quote.
- Every page → Request a Quote (primary conversion).

## Content cadence (post-launch)
- Publish 1–2 knowledge-centre articles/month targeting informational keywords.
- Expand product detail as verified specs become available.
- Add industry pages only where the portfolio genuinely supports the application.

## Do / Don't
- **Do** use natural Ahmedabad/Gujarat geo-context in copy.
- **Don't** keyword-stuff, invent specs, or imply manufacturing/partnerships.
